portfolio

A Portfolio is meant to represent a collection of individual Positions.

class portfolio.Order(id_number: int, symbol: str, quantity: float, order_type: portfolio.OrderType)

A dataclass that represents a market order.

class portfolio.OrderStatus(value)

An Enum holding a value for the status of an order (pending, completed, failed).

class portfolio.OrderType(value)

An Enum holding a value for an order either being a buy or a sell order.

class portfolio.Portfolio(alpaca_api, data_settings, starting_cash=10000, name=None)

A portfolio is simply a collection of individual positions.

add_existing_position(new_position)

Adds an initialized Position object to the Portfolio.

Parameters

new_position – The incoming and already initialized Position object to be added to the Portfolio.

Raises

  • ValueError – When there is a Position object for the same symbol as a Position already in the Portfolio.

  • TypeError – When new_position is not of type machine.position.Position.

cancel_order(id_of_order_to_cancel)

Removes the order with the given ID from the order queue.

Parameters

id_of_order_to_cancel – The ID of the order that needs to be cancelled.

Returns

True if the order was successfully removed from the queue, False otherwise.

contains_position(symbol)

Determines if any Position in the Portfolio uses the given symbol.

Parameters

symbol – A string for the market symbol of this position (i.e. “AAPL” or “GOOG”).

Returns

A bool for if the given symbol was found in any Position in the Portfolio.

create_new_position(symbol, initial_quantity)

Create a new Position from the provided args and add it to the Portfolio.

Parameters

  • symbol – A string for the market symbol of this position (i.e. “AAPL” or “GOOG”).

  • initial_quantity – The quantity of this asset that should be held when this instance is finished being constructed.

Returns

A newly created Position object that has been added to self.positions

delete_empty_positions()

Delete all positions in the portfolio that have 0 quantity currently held.

get_position(symbol)

Gets the Position object from the Portfolio with the given symbol

Parameters

symbol – A string for the market symbol of this position (i.e. “AAPL” or “GOOG”).

Returns

The Position object with the given symbol from within this Portfolio, returns None if no such Position exists.

place_order(symbol, quantity, order_type=OrderType.BUY)

Places an order to buy or sell some quantity of an asset. Adds the order to an order queue and does not directly execute the order. That is done by process_pending_orders().

Parameters

  • symbol – A string for the market symbol of this position (i.e. “AAPL” or “GOOG”).

  • quantity – The quantity of the asset to be bought or sold.

  • order_type – A value from the enum OrderType that represents if the order is a buy or a sell order. Defaults to OrderType.BUY.

Raises

ValueError – When the value passed into order_type is not in the enum OrderType.

Returns

The unique order ID number for the order being created.

total_value()

Returns

The total value of the Portfolio, including Positions and total cash.
